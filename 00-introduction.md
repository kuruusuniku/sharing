# 0. はじめに

## あの日の衝撃を、覚えていますか

2022年の終わり。ChatGPTが公開されたあの日。

私はiPhoneのSiriやAlexaに慣れていました。「今日の天気は？」と聞けばまあまあ答える。でも少し込み入ったことを聞くと、すぐに「すみません、よく分かりません」。AIってそういうものだと思っていた。

ChatGPTを触った瞬間、世界がひっくり返りました。

**え？ ヤバくないこれ。的確に応答してくるんだけど？**

コードを書いてと言えば書く。文章を直してと言えば直す。比喩が通じる。冗談が通じる。「ここをもう少しカジュアルに」と言えば、ちゃんとカジュアルになる。

SiriやAlexaとは、次元が違った。

そしてそこからの進化が、とてつもなく速かった。GPT-4が出て、Claude が出て、コードが書けるようになり、エージェントが動くようになり——こんなにも世界を変えるなんて、あの日は想像もしていなかった。

---

## なぜ、AIは突然「知性的」になったのか？

正直に言います。私もずっとこの問いを抱えていました。

大学でAIを専攻していた人間として（詳しくは[著者について](about)）、余計に不思議だった。私が研究していた頃のAIは、もっと素朴で、もっと限界がはっきりしていた。「自然言語処理の先にAIの未来はないよ」と言われて断念した過去すらある。

それが今、**言語モデルが知性の最前線にいる。**

なぜ「次の単語を予測する」だけの仕組みが、ここまで知的に振る舞えるのか。その答えは[13章](13-llm-internals)で詳しく解説しますが、さわりだけ書いておきます。

> LLMは次の単語を正確に予測しようとする過程で、「副産物」として世界のモデルを内部に構築する。天気を予測するために気象学が必要なように、次の単語を予測するために、現実世界の構造そのものを理解する必要がある。

「王様」から「男性」を引いて「女性」を足すと「女王」になる——誰も教えていないのに、AIの内部空間にこの関係性が自然と立ち上がる。言語と確率によって、この現実世界をかなりの精度で記述できてしまう。それが、AIが「分かっている」ように見える本当の理由です。

興味があれば、[13章「LLMを支える仕組み」](13-llm-internals)へ。ここでは先に、**このAIをどう使いこなすか**の話を進めます。

## AIの進化が閾値を超えた

AI開発はいま急速に進んでおり、2大巨頭は **Anthropic** と **OpenAI** です。

| 企業 | ツール | 最新モデル |
|------|--------|-----------|
| **Anthropic** | Claude Code | Opus 4.7 |
| **OpenAI** | Codex | GPT-5.5 |

Google（Gemini）のモデルはまだ実務で使える閾値を超えておらず、仕事で本格活用するなら基本的にこの2社のツールが選択肢になります。

## 日常利用と仕事利用の違い

日常的な相談ごと（調べもの、文章作成、アイデア出し）であれば ChatGPT や Gemini でも十分です。

ただし、**仕事として成果物を出す**となると話が変わります。コードを書く、資料を作る、タスクを管理する——こうした業務にはAnthropicかOpenAIのツールが必要です。

## なぜ Claude Code なのか

Claude Code と Codex はどちらも優れていますが、**現時点でのメインストリームは Claude** です。

まずは **Claude Desktop（デスクトップアプリ）** から始めるのもアリです。チャット形式で Claude と対話でき、日常業務にすぐ使えます。

ただし、AIを高度に活用する——ファイル操作、コマンド実行、複数タスクの並列処理——には **Claude Code（CLI）** が最適です。この資料では Claude Code を中心に解説します。

## プランについて

| プラン | 月額 | 所感 |
|--------|------|------|
| Pro | $20 | 入門には十分。ただし利用制限にすぐ当たる |
| **Max** | **$100** | 満足に使うにはこのプランが必要 |
| Max 20x | $200 | ヘビーユーザー向け。制限がさらに緩和 |
| Team | $30/人 | 個人の Max プランの方がコスパが良い |

正直なところ、Max $100 プランに入らないと仕事で満足に使えない面はあります。Team プランは管理機能が加わる分、個人の利用量としては Max の方がお得です。

## 賢いモデルだけでは足りない

AIモデルがどれだけ賢くても、それ単体では力を発揮しきれません。

**モデルの能力を引き出すには、それを組み込む「仕組み」が必要です。**

- どういう順番でタスクを処理するか
- 複数のAIをどう協調させるか
- 成果物の品質をどうチェックするか

これが、後の章で紹介する **ワークフロー**（[2章](02-agent-patterns)）や **オーケストレーションツール**（[4章](04-custom-orchestration)）と呼ばれるものです。

この資料を通じて、AIを「使う」だけでなく「仕組みとして活用する」方法を共有できればと思います。
