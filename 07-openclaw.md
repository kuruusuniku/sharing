# 7. OpenClaw — 個人用AIアシスタント

![OpenClaw 概要](assets/images/ch07-openclaw.svg)

ここまでの章では、コーディングや業務タスクにフォーカスしてきました。この章では視点を変えて、**日常生活全体をAIでアシストする**ツールを紹介します。

私も使っていますが、正直まだ使いこなせていません。一緒に育てていきましょう。

### たとえ話：眠らない個人秘書

OpenClaw は、**あなた専用の、眠らない秘書**のようなものです。

Siri や Google Assistant は「ちょっと聞いたら答えてくれる受付」。OpenClaw は違う。あなたの好みを覚え、あなたのアプリを操作し、あなたのルーティンを自動でこなす。しかも、どのLLMを「脳」にするか、どのサービスと繋ぐかは、全部あなたが決められる。

あなたもこう思ったことありませんか？——「Siri はいまいち融通が利かない。かといって自分でスクリプト書くのも面倒」。OpenClaw はその中間にいるツールです。自然言語で指示できるけど、裏側は自分で自由にカスタマイズできる。

## OpenClaw とは

[OpenClaw](https://openclaw.ai/) は、オープンソースの個人用AIアシスタントです。自分のPC上で動作し、複数のチャットアプリ・サービスと連携して、日常のあらゆるタスクを自動化できます。

**一言でいうと：** 「自分専用の Siri / Google Assistant を、好きなLLMで、好きなサービスと繋げて作れる」ツールです。

- GitHub: [github.com/openclaw/openclaw](https://github.com/openclaw/openclaw)
- ドキュメント: [docs.openclaw.ai](https://docs.openclaw.ai)
- コミュニティ: [Discord](https://discord.gg/clawd)

## 他のツールとの違い

| | ChatGPT / Claude Desktop | OpenClaw |
|--|--------------------------|----------|
| **LLM** | 固定（各社のモデル） | 自由に選べる（Claude, GPT, ローカルLLM等） |
| **データ** | クラウドに送信 | ローカルに保存 |
| **連携先** | 限定的 | 自由に拡張可能 |
| **カスタマイズ** | プロンプトのみ | ワークフロー全体を自由に設計 |
| **料金** | サブスク | LLMのAPI費用のみ |

## 主な機能

| 機能 | 説明 |
|------|------|
| **マルチチャット連携** | Telegram, Discord, Slack, LINE 等を一元管理 |
| **ワークフロー自動化** | 「毎朝ニュースを要約して送る」等のルーティン |
| **ファイル処理** | PDF要約、画像分析、音声文字起こし |
| **記憶** | 会話の文脈を長期記憶として保持 |
| **ツール連携** | カレンダー、メール、Notion 等への操作 |

## セットアップ

```bash
# Docker で起動（推奨）
docker compose up -d

# 設定
# http://localhost:3000 にアクセスして初期設定
```

必要なもの：
- Docker
- LLMのAPIキー（Claude, GPT 等）
- 連携したいサービスのAPIキーやBot Token

## 活用アイデア

- **朝の情報収集自動化** — ニュース、天気、今日の予定を毎朝まとめて送信
- **メッセージの自動振り分け** — 複数チャットの通知を重要度で整理
- **議事録の自動作成** — 会議音声を文字起こし→要約→関係者に共有
- **学習サポート** — 読書メモの整理、復習リマインダー

### 「育てる」感覚

OpenClaw の面白いところは、**使い込むほど賢くなる**ところです。記憶機能があるので、あなたの好みや癖を学習していく。最初は「天気教えて」くらいしか使わなかったのが、数週間後には「今日の予定を見て、移動時間を考慮して、雨なら傘リマインダーを送って」まで自動化されている。

ペットを育てる感覚に近いかもしれません。最初は何もできないけど、一緒に過ごすうちにどんどん気が利くようになる。

## Q&A — よくある疑問

### Q: プライバシーは本当に安全？ データはどこに行く？

**A:** OpenClaw はローカルで動作するため、データは基本的にあなたのマシン内に留まります。ただし、LLM の API を使う場合は、プロンプト（あなたの指示とコンテキスト）が API プロバイダー（Anthropic, OpenAI 等）のサーバーに送信されます。完全にローカルで完結させたい場合は、LM Studio や Ollama でローカル LLM を使う構成も可能です。連携先サービス（Slack, Gmail 等）との通信は直接行われ、OpenClaw のサーバーを経由しません。

### Q: 既存のワークフローを壊さない？ 導入リスクは？

**A:** OpenClaw は「追加」であって「置き換え」ではありません。既存のツール（Slack, カレンダー, メール等）はそのまま使い続けます。OpenClaw はそれらを「横から見て、自動化する」だけ。気に入らなければ Docker を止めるだけで元に戻ります。段階的に機能を足していけるので、いきなり全部を任せる必要はありません。まず「朝の天気通知」だけから始めて、慣れたら範囲を広げる、というアプローチがおすすめです。

### Q: 育てるのにどれくらいかかる？ すぐ使える？

**A:** 正直に言うと、初期セットアップに半日、基本的な自動化が動くまでに1-2日、「自分好みの秘書」になるまでに1-2週間です。即座に価値が出るツールではありません。「毎日少しずつ教える」タイプのツールです。ただし、一度設定したワークフローは永続的に動き続けるので、投資対効果は時間とともに上がっていきます。「1時間かけて設定したものが、毎日5分節約してくれる」の積み重ねです。
